## Что измеряем

| Метрика | Смысл |
|---------|--------|
| **Concurrent connections** | Одновременные WebSocket (SignalR) клиенты |
| **Messages / second** | Исходящих/входящих сообщений Hub в агрегате |
| **Latency (p50 / p95 / p99)** | Время от отправки до получения на клиенте |
| **CPU / память** | По процессу `dotnet` и Redis |
| **Ошибки и reconnect** | Процент обрывов, таймаутов, 5xx |

Целевые порядки величин для **демо** (не SLA продакшена): десятки тысяч соединений и десятки тысяч msg/s при устойчивой задержке и без деградации Redis — как ориентир для настройки и сравнения конфигураций.

## Инструменты

Примеры инструментов, совместимых с .NET и WebSocket:

| Инструмент | Назначение |
|------------|------------|
| **k6** + WebSocket extension | Сценарии: connect → join → receive/send в цикле |
| **bombardier / wrk** | HTTP-only прогоны (менее релевантно для чистого WS, полезно для health/metrics) |
| **Кастомный .NET load generator** | Полный контроль над SignalR client protocol |

На стороне сервера полезно снимать:

- встроенные **ASP.NET Core / Kestrel** счётчики;
- **dotnet-counters** / **dotnet-trace**;
- метрики **Redis** (latency, ops/s, blocked clients).

---

## Сценарии

1. **Ramp-up:** постепенное увеличение соединений до целевого N (наблюдать стабильность).
2. **Steady broadcast:** один паблишер → группа из M подписчиков; измерять msg/s и задержку.
3. **Multi-instance:** два и более инстанса приложения + Redis backplane; убедиться, что сообщения доходят до клиентов на **разных** узлах.

Фиксируйте: версия .NET, конфигурация Kestrel, лимиты ОС, параметры Redis, число реплик.

## Интерпретация

- Рост **p99** при стабильном CPU часто указывает на **GC**, блокировки или очереди на Redis.
- Рост **blocked clients** в Redis — сигнал пересмотреть размер сообщений, частоту публикаций или топологию.
- При scale-out без backplane клиенты на разных узлах **не** увидят общие broadcast — это ожидаемое поведение, см. [ADR 0001](../adr/0001-use-signalr-with-redis-backplane.md).

Подробности запуска окружений: [setup.md](./setup.md).
